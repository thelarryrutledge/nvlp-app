No-Vendor Ledger Platform (NVLP) API for envelope budgeting and personal finance management.

API Architecture

The NVLP API uses a hybrid architecture with custom domain routing:

  • PostgREST Direct (db-api.nvlp.app): Fast CRUD operations for simple data access
  • Edge Functions (edge-api.nvlp.app): Complex business logic, validation, and advanced features

This dual-domain approach provides optimized routing and performance for different operation types.

Base URLs

  • Custom Domain (PostgREST): https://db-api.nvlp.app
  • Custom Domain (Edge Functions): https://edge-api.nvlp.app
  • Fallback PostgREST: https://qnpatlosomopoimtsmsr.supabase.co/rest/v1
  • Fallback Edge Functions: https://qnpatlosomopoimtsmsr.supabase.co/functions/v1

Authentication

All endpoints require JWT authentication using the Authorization: Bearer <token> header. Tokens are obtained through the authentication endpoints and expire after a configurable period.

Authentication Headers

  • Edge Functions: Authorization: Bearer <token>
  • PostgREST: Authorization: Bearer <token> + apikey: <anon_key>

Data Security

  • Row Level Security (RLS) ensures users can only access their own data
  • All sensitive operations are logged in the audit trail
  • Multi-tenant architecture with complete data isolation
