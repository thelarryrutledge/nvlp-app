NVLP API Documentation

Virtual Envelope Budget Application Programming Interface

Overview

The NVLP (Virtual Envelope Budget) API provides a complete backend solution for envelope-based budgeting applications. Built on Supabase with PostgREST and Edge Functions, it offers real-time data synchronization, robust security, and comprehensive financial management capabilities.

🎯 Envelope Budgeting

Complete envelope-based budgeting system with support for regular budgeting, savings goals, and debt management. Each envelope can have custom notifications and tracking.

💳 Smart Debt Tracking

Specialized debt envelopes that automatically track remaining balances, minimum payments, and due dates. Debt payments reduce both envelope and debt balances.

🔄 Real-time Sync

Built on Supabase with real-time subscriptions, ensuring all connected clients stay synchronized with the latest financial data.

🛡️ Security First

Row Level Security (RLS) ensures users can only access their own data. JWT-based authentication with Supabase Auth integration.

📊 Automated Calculations

Cached financial state and automatic balance calculations ensure fast performance even with large transaction volumes.

🔧 Hybrid Architecture

PostgREST for fast CRUD operations and Edge Functions for complex business logic, giving you the best of both worlds.

Technology Stack

Database: PostgreSQL with Supabase
API Layer: PostgREST for CRUD operations
Business Logic: Supabase Edge Functions
Authentication: Supabase Auth (JWT)
Security: Row Level Security (RLS)
Real-time: Supabase Realtime

Key Features

  • Envelope Types: Regular, Savings, and Debt envelopes with specialized behaviors
  • System Categories: Protected categories for Savings, Loans, Credit Cards, and Debt
  • Automatic Debt Tracking: Debt payments automatically reduce both envelope and debt balances
  • Flexible Notifications: Envelope-level notifications for low/high balances and dates
  • Income Scheduling: Flexible income frequency with custom day scheduling
  • Cached Performance: Pre-calculated totals and balances for fast queries
  • Soft Deletes: Transaction history preservation with soft delete functionality
  • Multi-user Support: Complete user isolation with RLS policies

Getting Started

  1. Authentication: Obtain a JWT token from Supabase Auth
  2. Create Budget: Set up your first budget container
  3. Add Categories: System categories are auto-created, add custom ones as needed
  4. Create Envelopes: Set up envelopes for different budget purposes
  5. Record Transactions: Add income, expenses, and transfers
  6. Track Progress: Monitor your financial state through the API
