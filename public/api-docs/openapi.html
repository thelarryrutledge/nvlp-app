Error Responses

All endpoints may return the following error responses:

  • 400 Bad Request - Invalid request data
  • 401 Unauthorized - Authentication required or invalid
  • 404 Not Found - Resource not found
  • 500 Internal Server Error - Server error

Special Behaviors

Envelope Type Validation

The API enforces envelope type and category compatibility:

  • Savings envelopes must be in the "Savings" system category
  • Debt envelopes must be in debt-related system categories (Loans, Credit Cards, or Debt)
  • Regular envelopes cannot be in system categories

Debt Payment Handling

When creating debt_payment transactions:

  • The transaction amount is deducted from the envelope's current_balance
  • The transaction amount is also deducted from the envelope's debt_balance
  • This behavior is automatic and handled by database triggers

System Categories

System categories (Savings, Loans, Credit Cards, Debt) are automatically created for new budgets and cannot be modified or deleted through the API.

This API reference reflects the current implementation including envelope types, system categories, and debt handling functionality.
